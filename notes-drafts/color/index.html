Давайте поговорим о первых — подобранных цветах. Что мы о них знаем?

Цвет точек, которые вы видите на экране хранятся как три значения цветовых каналов красного, зелёного и синего цветов. Это модель RGB.

< Есть RGB, XYZ, LMS — трёхкомпонентые цветовые модели, хранящие значения для трёх видов наших колбочек. habrahabr.ru/post/209738/ Это лучше чем ничего, такая модель хранит достаточно данных для монитора, адаптированного для нашего глаза. Этого недостаточно для Mantis Shrimp, у неё 16 видов колбочек: http://theoatmeal.com/comics/mantis_shrimp

Для вывода на экран цвет нужно представить в ней. Вычислять, смешивать и хранить цвета мы можем в любой цветовой модели, главное — выдать RGB-цвет для отображения.

Каждый цветовой канал имеет 256 градаций, 2^8. Для хранения такого количества значений нужно 8 бит на канал или 24 бита на цвет. Это число называют глубиной цвета. 24 бита — стандартная современная глубина RGB цвета. Некоторые устройства поддерживают 30, 36 или 48 бит, но ваш браузер показывает и различает только 24. Это относится ко всем веб-технологиям: HTML, CSS, SVG, Canvas работают с 24-битным цветом.

Посмотрим на RGB-кубик.

В кубике 16 777 216 цветов. Для начала, мы можем уменьшить количество, выкидываем промежуточные значения: http://jsfiddle.net/phoq7qe4/1/

Есть кубик. RGB, непонятно. < сайт где люди угадывают цвет по хексу > Мы хотим других метрик! Яркость, контраст, насыщенность, тон! Но хотим предсказуемую метрику, с известным диапазоном.

Как переставить точки кубика, чтобы что-то получить? Повернём кубик, чёрный внизу, белый сверху. Яркость! Увы, потеряли предсказуемость. Берём полярные координаты и доводим до цилиндра. Сначала до двойного конуса.

C = M — m L = (M + m) / 2 S = 0? C = 0 S = C / (1 — |2L — 1|)

Правда, клёвая формула для яркости? Вместо L=1/3 для RGB и L=2/3 для CMY мы получаем 0.5! Супер. В HSV вообще получаем 1.

Итого: построили удобную модель мы получили совершенную неадекватную L и ограниченно применимую S.
